Overview

What is Webpack?

Webpack is a powerful module bundler for JavaScript applications. It takes modules with dependencies and generates static assets representing those modules.

Why Use Webpack?

Key Benefits

Webpack offers numerous benefits including module bundling, code splitting, asset management, and more. It helps optimize your web applications by managing dependencies and minimizing the code size.

When do you need Webpack , error "can not use import statement outside a module"

The issue you're encountering with the import statement outside a module is because you're trying to use import directly in a script that's not being treated as a module. To resolve this for a browser environment (like GitHub Pages), where you cannot directly use ES6 module syntax without additional setup, you can use WebPack

Installation

Installing Webpack

To install Webpack, you need Node.js and npm installed on your system. Run the following command to install Webpack and Webpack CLI globally:

npm install -g webpack webpack-cli

Alternatively, you can install them as dev dependencies in your project:

npm install --save-dev webpack webpack-cli

Basic Setup

Project Structure

Create a basic project structure with the following files:

.
├── dist
│   └── index.html
├── src
│   └── index.js
└── webpack.config.js

webpack.config.js

Here's a basic configuration for Webpack:

const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  mode: 'development'
};

Loaders

Using Loaders

Loaders allow you to preprocess files as you import or load them. To use Babel for transpiling JavaScript, install the necessary packages:

npm install --save-dev babel-loader @babel/core @babel/preset-env

webpack.config.js

Add the following module rules to your Webpack configuration:

module: {
  rules: [
    {
      test: /\.js$/,
      exclude: /node_modules/,
      use: {
        loader: 'babel-loader',
        options: {
          presets: ['@babel/preset-env']
        }
      }
    }
  ]
}

Plugins

Using Plugins

Plugins are used to perform a wider range of tasks like bundle optimization, asset management, and injection of environment variables.

Installing HTML Webpack Plugin

Install the HTML Webpack Plugin:

npm install --save-dev html-webpack-plugin

webpack.config.js

Add the plugin to your Webpack configuration:

const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  // ... other configurations ...
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html'
    })
  ]
};

Dev Server

Setting Up Webpack Dev Server

The Webpack Dev Server provides you with a simple web server and the ability to use live reloading.

Installation

npm install --save-dev webpack-dev-server

webpack.config.js

Add the following configuration to your Webpack config:

devServer: {
  contentBase: './dist',
  hot: true
}

Package.json

Add a script to your package.json to start the dev server:

"scripts": {
  "start": "webpack serve --open"
}

Advanced Configuration

Code Splitting

Code splitting allows you to split your code into various bundles which can then be loaded on demand or in parallel.

webpack.config.js

module.exports = {
  // ... other configurations ...
  optimization: {
    splitChunks: {
      chunks: 'all',
    },
  },
};
