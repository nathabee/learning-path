Git Admin Tasks

  • Update Username
  • Update Email
  • Update Remote GitHub Reference
  • Generate SSH Key
  • Add SSH Key to GitHub
  • Update Authentication Method

1. Update Git Username

# Set the Git username globally
            git config --global user.name "Your Name"
            
            # Verify the update
            git config --global user.name

2. Update Git Email

# Set the Git email globally
            git config --global user.email "your.email@example.com"
            
            # Verify the update
            git config --global user.email

3. Update Remote GitHub Reference

# Change the remote URL for a repository
            git remote set-url origin https://github.com/user/repo.git
            
            # Verify the update
            git remote -v

4. Generate SSH Key

Generate a new SSH key pair for authentication:

# Generate a new SSH key pair
            ssh-keygen -t ed25519 -C "your.email@example.com"
            
            # Start the ssh-agent in the background
            eval "$(ssh-agent -s)"
            
            # Add your SSH private key to the ssh-agent
            ssh-add ~/.ssh/id_ed25519

5. Add SSH Key to GitHub

To add your SSH key to your GitHub account:

# Copy the SSH key to your clipboard (Linux/macOS)
            cat ~/.ssh/id_ed25519.pub | pbcopy
            
            # Copy the SSH key to your clipboard (Windows)
            type ~/.ssh/id_ed25519.pub | clip

Then, go to your GitHub account settings:

  1. Navigate to Settings > SSH and GPG keys.
  2. Click on New SSH key.
  3. Paste the copied SSH key into the Key field and give it a descriptive title.
  4. Click Add SSH key.

6. Update Authentication Method

To switch to SSH authentication for a repository:

# Change the remote URL to use SSH
            git remote set-url origin git@github.com:user/repo.git
            
            # Verify the update
            git remote -v

Git Commands for Monorepository

  • Create Repository
  • Clone Repository
  • Add New Project
  • Work on a Project
  • Commit Changes
  • Clone Monorepo
  • Directory Structure

1. Create the Monorepository on GitHub

Create the Monorepository:

Go to GitHub and create a new repository called mono-repo.

2. Clone the Monorepository Locally

# Clone the monorepository to your local machine
                git clone https://github.com/user/mono-repo
                cd mono-repo

3. Add a New Project to the Monorepository

# Create a directory for the new project
                mkdir projects/new-project
                cd projects/new-project
                
                # Initialize a new Git repository
                git init
                
                # Add initial files and commit
                echo "# New Project" > README.md
                git add .
                git commit -m "Initial commit for new project"
                
                # Push the new project to the monorepository
                git remote add origin https://github.com/user/mono-repo
                git push origin main

4. Work on a Project

Navigate to the project's directory, make changes, and commit those changes within the monorepository.

# Navigate to the project directory
                cd projects/new-project
                
                # Make changes, add them, and commit
                git add .
                git commit -m "Make changes to the project"
                
                # Push the changes to the monorepository
                git push origin main

5. Commit Changes in the Monorepository

After making changes in a project, you need to commit the changes in the monorepository.

# Navigate back to the monorepository root
                cd ../../
                
                # Add and commit the changes
                git add projects/new-project
                git commit -m "Update new project"
                git push origin main

6. Clone the Monorepository

When someone clones the monorepository, they get all the projects within it.

# Clone the monorepository
                git clone https://github.com/user/mono-repo
                cd mono-repo

Example Directory Structure with .gitignore

Your project directory might look like this:

/path/to/mono-repo
                |-- .gitignore          # Root .gitignore
                |-- /projects
                    |-- /new-project
                        |-- .gitignore  # Project .gitignore

Git Commands for Project with Submodules

  • Create Repositories
  • Clone Main Repository
  • Add Submodule
  • Initialize Submodules
  • Work on Submodule
  • Commit in Main Repo
  • Clone with Submodules
  • Directory Structure

1. Create the Main Repository and Submodule Repositories on GitHub

Creating Repositories:

Create the Main Repository:

Go to GitHub and create a new repository called main-repo.

Create the Submodule Repository:

Similarly, create another repository called new-submodule.

2. Clone the Main Repository Locally

# Clone the main repository to your local machine
git clone https://github.com/user/main-repo
cd main-repo

3. Add a Submodule to the Main Repository

# Add the submodule to the main repository
git submodule add https://github.com/user/new-submodule path/to/new-submodule

# Commit the addition of the submodule
git commit -m "Add new submodule"

4. Initialize and Update Submodules

After adding submodules, you need to initialize and update them to ensure they are properly integrated into the main repository.

# Initialize and update submodules
git submodule update --init --recursive

5. Work on a Submodule

To work on a submodule, navigate to the submodule's directory, make changes, and commit those changes within the submodule.

# Navigate to the submodule directory
cd path/to/new-submodule

# Make changes, add them, and commit
git add .
git commit -m "Make changes to submodule"

# Push the changes to the submodule repository
git push origin main

6. Commit Changes in the Main Repository

After making changes in the submodule, you need to commit the updated submodule reference in the main repository.

# Navigate back to the main repository
cd ../../

# Add and commit the updated submodule reference
git add path/to/new-submodule
git commit -m "Update submodule reference"
git push origin main

7. Initialize Submodules When Cloning the Main Repository

When someone clones the main repository, they need to initialize the submodules.

# Clone the main repository
git clone https://github.com/user/main-repo
cd main-repo

# Initialize and update submodules
git submodule update --init --recursive

Example Directory Structure with .gitignore

Your project directory might look like this:

/path/to/main-repo
|-- .gitignore          # Root .gitignore
|-- /path/to/new-submodule
    |-- .gitignore      # Submodule .gitignore
