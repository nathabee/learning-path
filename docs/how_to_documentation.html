Overview

Presentation

This documentation is static and available on GitHub Pages. The goal is to trace my learning path, so I can easily revisit an old theme after a long time without practicing it.

We could see it like a wiki. However, I am more focused on presenting the themes in a way that is useful for me, without spending too much time creating this documentation.

History

This documentation is made on the go while I am learning to code. At the beginning of my learning journey, I was not using a document archive, and the memories of my journey faded away. This is like taking pictures while on a trip and archiving them quickly, without effort.

Step by step document creation

  • Clone the repository
  • Use the source code of docs/how_to_add_doc.html template from this GitHub repository
  • Open a session of ChatGPT and explain the theme that you want to explain
  • Copy-paste the code generated by ChatGPT
  • Copy the HTML created by ChatGPT into docs
  • In the repository directory, run ./prepare.github.sh (creates styles.css, bundle.js, and metadata.js)
  • Test the file locally
  • When the test is successful, archive the new docs on GitHub
  • Ensure that the file is deployed on GitHub Pages

Publish

1. Prepare for GitHub

We call a script that will create:

  • bundle.js (with webpack)
  • metadata.js (extract information from HTML files for navigation and search)
  • styles.css (automatically generated with tailwind.css)
./prepare-github.sh

2. Send to GitHub

git add .
git status
git commit -m "docs add tra la la"
git push origin main

3. Deploy on GitHub Pages

Check that the code is published. To publish on GitHub Pages, see the reference: learning-path/docs/how_to_deploy_github_page.html
