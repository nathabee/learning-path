How to Deploy on GitHub Pages

Steps to Deploy on GitHub Pages

1. Create or Update Your Repository

Ensure your HTML files are in a GitHub repository. If they are under a docs directory, you can use that directory directly for GitHub Pages.

2. Configure GitHub Pages

  • Go to your repository on GitHub.
  • Click on the Settings tab.
  • Scroll down to the Pages section.
  • In the Source dropdown, select the branch you want to use (usually main or master).
  • If your HTML files are in the docs folder, select /docs folder.
  • Click Save.

3. Access Your Site

After saving, GitHub will start building your site. Once the build is complete, you will see a URL where your site is published, usually in the format https://username.github.io/repository-name/.

Detailed Explanation

Single Page Limitation

GitHub Pages serves static content, which means it is suitable for hosting HTML, CSS, and JavaScript files without server-side processing. There is no inherent single-page limitation; you can host multi-page websites as long as they are purely static. Here’s how it works:

  • Static Content: HTML files, CSS files, images, and JavaScript files can be hosted and will be served directly to users' browsers.
  • Navigation: Your HTML files can link to each other using relative or absolute URLs. For example, if you have an index.html and an about.html, you can link between them like this: 
    <!-- index.html -->
<a href="about.html">About</a>

<!-- about.html -->
<a href="index.html">Home</a>

Relative Links

Ensure your links are relative to the current document to make navigation between pages smooth. Here’s an example of how you can structure your links:

Directory Structure:

docs/
├── index.html
├── about.html
├── contact.html

Navigation Example:

<!-- index.html -->
<a href="about.html">About</a>
<a href="contact.html">Contact</a>

<!-- about.html -->
<a href="index.html">Home</a>
<a href="contact.html">Contact</a>

<!-- contact.html -->
<a href="index.html">Home</a>
<a href="about.html">About</a>

Advanced Configuration

If you need more advanced features, such as custom domains or Jekyll processing, GitHub Pages supports those as well. For a basic static site, the steps above are sufficient.

Summary

  1. Ensure your repository is public change from private to public in settings, change visibility
  2. Ensure your HTML files are in a repository (preferably in a docs folder if you choose to use that as the source).
  3. Configure GitHub Pages in the repository settings to use the main branch and the docs folder (if applicable).
  4. Access your website via the provided github.io URL.

This setup will allow you to navigate between multiple HTML files hosted on GitHub Pages without any issues, as long as they are all static files.
